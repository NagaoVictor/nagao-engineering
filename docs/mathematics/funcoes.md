\# Funções



\## O que é?



Uma função é uma regra que transforma uma entrada em exatamente uma saída.



Ela recebe um valor, realiza algum processamento e devolve um único resultado.



Na matemática, funções descrevem transformações. Na computação, elas aparecem como funções, métodos, APIs, algoritmos e até modelos de Machine Learning.



\---



\## Exemplo



Imagine um sistema que calcula o dobro de um número.



A regra é:



```text

f(x) = 2x

```



Agora responda:



\- Qual é a saída para `3`?

\- Qual é a saída para `7`?

\- Qual é a saída para `10`?



\---



\## Como resolver



Uma função aplica sempre a mesma regra.



Neste caso:



```text

f(x) = 2x

```



\### Entrada: 3



Substituímos `x` por `3`.



```text

f(3) = 2 × 3



f(3) = 6

```



\---



\### Entrada: 7



```text

f(7) = 2 × 7



f(7) = 14

```



\---



\### Entrada: 10



```text

f(10) = 2 × 10



f(10) = 20

```



Observe que cada entrada produz exatamente uma saída.



```text

3  →  6



7  → 14



10 → 20

```



Essa é a característica fundamental de uma função.



\---



\## O que \*\*não\*\* é uma função?



Imagine a seguinte situação:



```text

3 → 6



3 → 9

```



A mesma entrada possui duas saídas diferentes.



Isso \*\*não\*\* é uma função.



Uma função pode fazer várias entradas chegarem na mesma saída, mas \*\*nunca\*\* permitir que uma única entrada tenha duas saídas.



\---



\## Conceitos importantes



\### Domínio



É o conjunto de todas as entradas possíveis.



Exemplo:



```text

{1,2,3,4,5}

```



São todos os valores que podemos fornecer para a função.



\---



\### Imagem



É o conjunto das saídas realmente produzidas.



Para:



```text

f(x)=2x

```



e domínio



```text

{1,2,3}

```



A imagem será



```text

{2,4,6}

```



\---



\### Contradomínio



É o conjunto onde as saídas \*\*podem\*\* estar.



Nem sempre toda a imagem ocupa o contradomínio inteiro.



\---



\## O que aprender com isso



Uma função sempre responde à pergunta:



> \*\*"Se eu fornecer esta entrada, qual será exatamente a saída?"\*\*



Ela representa uma transformação previsível.



\---



\## Onde aparece na computação?



\- Funções (`function`)

\- Métodos (`method`)

\- APIs

\- Machine Learning

\- Criptografia

\- Processamento de imagens

\- Processamento de sinais

\- Compiladores

\- Sistemas embarcados



\---



\## Resumo



| Conceito | Ideia |

|----------|-------|

| \*\*Função\*\* | Regra que transforma uma entrada em uma única saída. |

| \*\*Entrada\*\* | Valor fornecido para a função. |

| \*\*Saída\*\* | Resultado produzido pela função. |

| \*\*Domínio\*\* | Todas as entradas permitidas. |

| \*\*Imagem\*\* | Todas as saídas realmente obtidas. |

| \*\*Contradomínio\*\* | Conjunto onde as saídas podem existir. |



\---



\## Modelo mental



Imagine uma máquina.



```text

Entrada

&#x20;  │

&#x20;  ▼

┌──────────┐

│ Função   │

└──────────┘

&#x20;  │

&#x20;  ▼

Saída

```



Você coloca um valor na máquina.



Ela aplica uma regra.



Sempre devolve \*\*uma única resposta\*\* para aquela entrada.



É exatamente assim que uma função matemática funciona — e é exatamente assim que funções em linguagens de programação funcionam.
